    CampaignKit iOS SDK Release Notes

    Version 0.0.7: iOS 8 & Xcode 6+ support

    Version 0.0.3: Changes in accessing content from CKCampaign

    A new CKContent class has been added allowing full access to content fields, including custom attributes. This is a breaking API change from the previouis version of the SDK.

    Properties moved

    Previously, only the content HTML was accessible via the CKContent class. In addition, the title and message fields on CKCampaign have been moved to CKContent. This new way of accessing these fields are shown below:

    OLD:

    CKCampaign *_campaign;
...
NSString *htmlContent = _campaign.content;
NSString *alertTitle = _campaign.title;
NSString *alertMessage = _campaign.message;

    NEW:

    CKCampaign *_campaign;
...
NSString *htmlContent = _campaign.content.body;
NSString *alertTitle = _campaign.content.alertTitle;
NSString *alertMessage = _campaign.content.alertMessage;

    Attribute access

    Access to arbitrary content attributes (key/value pairs) set up in the web portal are now available on CKContent. They may be accessed as follows:

    CKContent *content = _campaign.content;
NSString *value = [content.attributes objectForKey:@"my_key"];
